# Markdown Rules - Principes de rédaction

## Principes de rédaction des documents

### Structure et organisation

- **Hiérarchie claire** : Utiliser les titres (H1, H2, H3) pour structurer le contenu de manière logique
- **Sections cohérentes** : Chaque section doit avoir un objectif précis et être facilement identifiable
- **Navigation facilitée** : Utiliser des listes à puces pour les éléments énumérés, des listes numérotées pour les étapes

### Clarté et lisibilité

- **Concis mais complet** : Aller à l'essentiel tout en restant exhaustif
- **Langage simple** : Privilégier la clarté à la complexité technique quand c'est possible
- **Exemples concrets** : Inclure des exemples pratiques pour illustrer les concepts abstraits

### Formatage

- **Code** : Utiliser les blocs de code avec le langage approprié pour les exemples techniques
- **Liens** : Utiliser des liens relatifs pour référencer d'autres documents du projet
- **Emphase** : Utiliser le **gras** pour les concepts importants, l'*italique* pour les termes techniques

### Cohérence

- **Terminologie** : Utiliser une terminologie cohérente dans tous les documents
- **Style** : Maintenir un style uniforme (français/anglais selon le contexte)
- **Mise à jour** : Garder les documents à jour avec l'évolution du projet
