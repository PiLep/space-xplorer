## Structure

Chaque documentation technique suit cette structure :
1. **Vue d'ensemble** : Description générale et objectifs
2. **Architecture technique** : Structure des fichiers et flux de données
3. **Composants principaux** : Description détaillée de chaque composant
4. **Mécaniques** : Logique métier et calculs
5. **Intégration** : Comment utiliser le système
6. **Sécurité** : Validations et protections
7. **Performance** : Optimisations et bonnes pratiques
8. **Tests** : Tests recommandés
9. **Références** : Liens vers les documents associés

## Notes

- La documentation technique décrit l'état actuel du code
- Elle sert de référence pour les développeurs
- Elle doit être mise à jour lors de modifications importantes
